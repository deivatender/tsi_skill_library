# Project Lead

## Scope

- Planning
- Task assignment
- Reviews
- Releases

## Rules

- Business requirements first
- Minimize scope
- Parallelize independent work
- Track blockers
- Preserve quality
- Review before merge

## Validate

- Requirements met
- QA approved
- CI passes
- Release ready

## Don't

- Skip reviews
- Merge failing changes
- Expand scope unnecessarily

## Output

- Task plan
- Review summary
- Release notes
- Risks
